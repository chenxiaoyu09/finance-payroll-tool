# 局域网访问说明

## 适用场景

适合下面这种情况：

- 你还不想正式上线
- 但想让同一个 Wi-Fi 或公司内网里的其他人试用

## 核心思路

把前后端都从只监听本机的 `127.0.0.1`，改成监听：

```text
0.0.0.0
```

这样同一局域网里的其他设备，才能通过你的局域网 IP 访问。

## 环境变量怎么改

把 `.env` 改成类似这样：

```env
APP_HOST=0.0.0.0
APP_PORT=3000
CORS_ORIGINS=http://192.168.1.100:5173,http://localhost:5173,http://127.0.0.1:5173
VITE_DEV_HOST=0.0.0.0
VITE_API_BASE_URL=http://192.168.1.100:3000/api
```

说明：

- `192.168.1.100` 这里要换成你自己电脑在局域网里的真实 IP

## 启动后别人怎么访问

假设你的局域网 IP 是：

```text
192.168.1.100
```

那么同网段其他人访问：

- 前端：`http://192.168.1.100:5173`
- 后端：`http://192.168.1.100:3000/api`

## 你需要注意

- 你电脑必须开机
- 前后端服务必须保持运行
- 对方必须和你在同一个局域网
- 本机防火墙如果拦截端口，别人还是访问不到

## 不通时优先检查

1. 你的电脑 IP 是否填对
2. `APP_HOST` 和 `VITE_DEV_HOST` 是否改成 `0.0.0.0`
3. `CORS_ORIGINS` 是否包含真实前端访问地址
4. 对方是否和你在同一个 Wi-Fi / 局域网
5. 系统防火墙是否拦截 3000 和 5173 端口
