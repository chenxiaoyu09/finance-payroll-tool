# 财务算薪系统公网部署执行清单

## 当前结论

截至 2026 年 6 月 24 日，这套系统还没有真正发布到公网。

当前只能在你自己的电脑上通过以下地址访问：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3000/api`

这两个地址都属于本地回环地址，只能你自己机器访问，别人无法直接打开。

## 别人要怎么访问

别人想访问，必须满足下面三个条件：

1. 前端页面部署到公网域名
2. 后端接口部署到公网域名或公网 IP
3. 数据库和文件存储可以被线上服务稳定访问

部署完成后，访问形式会变成：

- 前端：`https://payroll.your-domain.com`
- 后端：`https://api.your-domain.com/api`

这时别人只需要打开前端网址就能访问。

## 推荐上线方案

### 方案 A：最适合正式给别人用

- 前端：Vercel / Netlify / 腾讯云静态托管 / 阿里云 OSS 静态站点
- 后端：云服务器 Ubuntu 22.04 + Docker
- 数据库：阿里云 RDS MySQL / 腾讯云 MySQL
- 文件存储：先继续本地磁盘，后续升级 OSS / COS

优点：

- 稳定
- 可控
- 后续方便扩权限、日志、备份

### 方案 B：先快速试用

- 前端：Vercel 或 Netlify
- 后端：Railway / Render
- 数据库：Railway MySQL 或云数据库

优点：

- 上线更快
- 不用先买整台服务器

## 正式上线前你需要准备什么

### 必备信息

- 一个域名，或者至少一个公网 IP
- 一台云服务器，或一个可运行 Node.js 服务的平台账号
- 一个 MySQL 数据库
- 你准备对外给别人访问的前端域名
- 你准备对外提供接口的后端域名

### 推荐最小配置

- 服务器：2 核 4G
- 系统：Ubuntu 22.04
- Node：22.x
- MySQL：8.x
- Redis：7.x

## 上线时要改哪些配置

根目录 `.env` 里重点改这几个：

```env
NODE_ENV=production
APP_PORT=3000
CORS_ORIGINS=https://payroll.your-domain.com
VITE_API_BASE_URL=https://api.your-domain.com/api
DATABASE_URL=mysql://用户名:密码@数据库地址:3306/finance_payroll_prod
UPLOAD_DIR=./uploads
JWT_SECRET=请改成高强度随机字符串
JWT_EXPIRES_IN=7d
```

## 实际执行步骤

### 第一步：部署数据库

1. 创建生产库，例如 `finance_payroll_prod`
2. 导入初始化脚本：`infra/sql/init.sql`
3. 记录数据库地址、账号、密码

### 第二步：部署后端

1. 准备服务器或托管平台
2. 上传项目代码
3. 配置后端环境变量
4. 执行依赖安装
5. 执行 Prisma 生成
6. 构建并启动后端

参考命令：

```bash
cd apps/server
pnpm install
pnpm prisma:generate
pnpm build
node dist/main.js
```

后端成功后，应能通过公网访问：

```text
https://api.your-domain.com/api/health
```

### 第三步：部署前端

1. 配置 `VITE_API_BASE_URL`
2. 构建前端
3. 将 `dist` 部署到静态托管

参考命令：

```bash
cd apps/web
pnpm install
pnpm build
```

部署后别人访问：

```text
https://payroll.your-domain.com
```

### 第四步：联调验证

至少检查下面几项：

1. 前端页面能打开
2. 健康检查能连通
3. 能上传工资表
4. 能生成算薪预览
5. 能导出算薪草稿
6. 能导出异常名单
7. 能导出员工工资单

## 如果你现在就想给别人试用

当前还没公网地址，所以现在别人不能访问。

要让别人访问，必须至少完成下面其中一种：

- 真正部署到公网
- 使用内网穿透，把你本机临时暴露出去

但长期来看，我更建议你走正式部署，不要依赖你电脑一直开着。

## 你最终会给别人什么链接

正式上线以后，你给别人的是前端地址，不是后端地址。

例如：

- 给普通使用者：`https://payroll.your-domain.com`
- 后端接口地址仅供系统内部调用：`https://api.your-domain.com/api`
