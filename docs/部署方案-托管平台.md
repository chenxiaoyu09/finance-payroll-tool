# 部署方案二：前端托管 + 后端托管平台

## 适用场景

如果你想尽快给别人试用，且暂时不想自己维护服务器，这个方案更轻。

## 推荐组合

- 前端：Vercel / Netlify
- 后端：Railway / Render
- 数据库：Railway MySQL / 阿里云 RDS / 腾讯云 MySQL

## 最终访问方式

- 前端：`https://payroll-web.your-domain.com`
- 后端：`https://payroll-api.your-domain.com/api`

## 前端部署要点

前端构建前需要配置：

```env
VITE_API_BASE_URL=https://payroll-api.your-domain.com/api
```

构建命令：

```bash
pnpm build
```

输出目录：

```bash
dist
```

项目已提供 Vercel 配置：

- [`apps/web/vercel.json`](/Users/kristen/Documents/Codex/2026-06-23/niha/finance-payroll-tool/apps/web/vercel.json)

## 后端部署要点

后端环境变量至少要有：

```env
NODE_ENV=production
APP_PORT=3000
CORS_ORIGINS=https://payroll-web.your-domain.com
DATABASE_URL=mysql://user:password@host:3306/finance_payroll_prod
UPLOAD_DIR=./uploads
JWT_SECRET=replace_with_long_random_secret
JWT_EXPIRES_IN=7d
```

构建命令：

```bash
pnpm build
```

启动命令：

```bash
node dist/main.js
```

项目已提供 Railway 配置：

- [`apps/server/railway.json`](/Users/kristen/Documents/Codex/2026-06-23/niha/finance-payroll-tool/apps/server/railway.json)

## 这个方案的优点

- 上线更快
- 不需要自己运维太多服务器细节
- 适合先试用和演示

## 这个方案的限制

- 文件存储和本地磁盘路径要格外注意
- 某些平台对本地持久化不友好
- 后续最好升级成对象存储

## 推荐上线顺序

1. 先部署数据库
2. 再部署后端
3. 最后部署前端
4. 用前端页面完整跑一遍上传、预览、导出流程
