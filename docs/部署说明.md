# 财务算薪系统部署说明

## 现在别人能访问吗

截至 2026 年 6 月 24 日，还不能。

原因很简单：

- 现在前端地址是 `http://127.0.0.1:5173`
- 现在后端地址是 `http://127.0.0.1:3000/api`
- `127.0.0.1` 只代表你自己的电脑

所以目前只有你本机能访问，别人不能通过互联网直接打开。

如果要让别人访问，最终必须把系统部署到公网，并给别人一个前端网址，例如：

- `https://payroll.your-domain.com`

后端则部署成：

- `https://api.your-domain.com/api`

## 当前可用部署形态

当前项目已经支持两种使用方式：

- 本地开发运行
- 基于 Docker 的一键部署演示

如果是你自己开发联调，继续使用本地运行最方便。
如果是要给同事试用、给测试验收，建议先走 Docker 部署。

## 一、本地开发运行

### 1. 环境变量

根目录准备 `.env`：

```env
APP_PORT=3000
CORS_ORIGINS=http://127.0.0.1:5173,http://localhost:5173
VITE_API_BASE_URL=http://127.0.0.1:3000/api
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=finance_payroll_dev
DB_USER=root
DB_PASSWORD=123456
DATABASE_URL=mysql://root:123456@127.0.0.1:3306/finance_payroll_dev
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
UPLOAD_DIR=./uploads
```

### 2. 启动后端

```bash
cd apps/server
pnpm dev
```

### 3. 启动前端

```bash
cd apps/web
pnpm dev
```

### 4. 访问地址

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3000/api`

## 二、Docker 部署演示

### 1. 启动

在项目根目录执行：

```bash
docker compose up --build
```

### 2. 启动后访问

- 前端页面：`http://127.0.0.1:8080`
- 后端接口：`http://127.0.0.1:3000/api`

### 3. 当前容器说明

- `mysql`：工资数据与上传记录数据库
- `redis`：后续缓存与任务能力预留
- `server`：NestJS 后端服务
- `web`：Vite 构建后的前端页面

## 三、正式上线建议

如果你准备让别人长期使用，不建议继续停留在本地机或临时 Docker 演示。更推荐下面这套结构：

- 前端部署到静态托管平台
- 后端部署到长期在线的 Node 服务平台
- MySQL 使用云数据库
- 上传文件改成对象存储

推荐组合：

- 前端：Vercel / Netlify / 静态站点服务
- 后端：Railway / Render / 云服务器
- 数据库：阿里云 RDS / 腾讯云 MySQL / Railway MySQL
- 文件存储：OSS / COS / S3

## 四、正式上线前还建议补齐

当前系统已经可以演示上传、识别、预览、导出，但如果要给团队正式使用，建议继续补：

1. 登录与角色权限
2. 导入模板校验
3. 算薪规则配置化
4. 导出正式工资单
5. 历史版本比对
6. 操作日志
7. 文件云存储

## 五、推荐你接下来怎么做

如果你的目标是“别人现在就能访问”，接下来最实际的动作不是继续本地开发，而是：

1. 准备云服务器或托管平台账号
2. 准备数据库
3. 准备一个前端访问域名
4. 按线上环境变量模板配置生产环境
5. 部署前端与后端
