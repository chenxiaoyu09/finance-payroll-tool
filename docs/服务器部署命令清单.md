# 服务器部署命令清单

这份清单用于把财务算薪系统部署到一台云服务器上，让别人通过浏览器访问。

默认部署方式：

- 一台云服务器
- Docker Compose
- Nginx 统一对外暴露 `80` 端口
- 前端访问地址：`http://服务器IP`
- 后端接口地址：`http://服务器IP/api`
- 暂时不需要域名也能测试

## 1. 服务器建议

最低测试配置：

- 系统：Ubuntu 22.04 或 Ubuntu 24.04
- CPU：2 核
- 内存：4GB
- 硬盘：40GB
- 开放端口：`22`、`80`

如果后面正式给财务多人使用，建议升级到：

- CPU：2 核及以上
- 内存：8GB
- 硬盘：80GB 以上
- 绑定域名并配置 HTTPS

## 2. 登录服务器

把下面的 `服务器IP` 替换成你自己的公网 IP：

```bash
ssh root@服务器IP
```

如果云厂商提供的是普通用户，例如 `ubuntu`：

```bash
ssh ubuntu@服务器IP
```

## 3. 安装 Docker

```bash
sudo apt update
sudo apt install -y docker.io docker-compose-plugin git
sudo systemctl enable docker
sudo systemctl start docker
docker --version
docker compose version
```

## 4. 上传项目代码

推荐方式是先把项目放到 Git 仓库，然后在服务器拉取：

```bash
sudo mkdir -p /srv
sudo chown -R $USER:$USER /srv
cd /srv
git clone 你的项目仓库地址 finance-payroll-tool
cd /srv/finance-payroll-tool
```

如果暂时没有 Git 仓库，也可以把本机项目压缩上传到服务器的 `/srv/finance-payroll-tool`。

## 5. 准备生产环境变量

```bash
cd /srv/finance-payroll-tool
cp .env.production.example .env.production
```

编辑环境变量：

```bash
nano .env.production
```

没有域名、先用 IP 测试时，建议这样填：

```env
NODE_ENV=production
APP_PORT=3000
APP_HOST=0.0.0.0
CORS_ORIGINS=http://服务器IP
VITE_API_BASE_URL=/api

DB_HOST=mysql
DB_PORT=3306
DB_NAME=finance_payroll_prod
DB_USER=root
DB_PASSWORD=换成一个强密码
DATABASE_URL=mysql://root:换成一个强密码@mysql:3306/finance_payroll_prod

REDIS_HOST=redis
REDIS_PORT=6379
REDIS_PASSWORD=

UPLOAD_DIR=./uploads

JWT_SECRET=换成一串很长的随机字符串
JWT_EXPIRES_IN=7d
```

注意：

- `DB_PASSWORD` 和 `DATABASE_URL` 里的密码必须一致。
- `VITE_API_BASE_URL=/api` 表示前端和后端走同一个域名/IP，不需要额外配置跨域。
- 如果以后绑定域名，把 `CORS_ORIGINS` 改成 `https://你的域名`。

## 6. 启动生产服务

```bash
cd /srv/finance-payroll-tool
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build
```

第一次启动会下载镜像和安装依赖，时间会比较久。

## 7. 查看服务状态

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production ps
```

正常情况下应该看到这些服务：

- `finance-payroll-nginx`
- `finance-payroll-web`
- `finance-payroll-server`
- `finance-payroll-mysql`
- `finance-payroll-redis`

## 8. 查看日志

如果页面打不开，先看日志：

```bash
docker compose -f docker-compose.prod.yml --env-file .env.production logs -f nginx
docker compose -f docker-compose.prod.yml --env-file .env.production logs -f web
docker compose -f docker-compose.prod.yml --env-file .env.production logs -f server
docker compose -f docker-compose.prod.yml --env-file .env.production logs -f mysql
```

## 9. 验证接口

在服务器上执行：

```bash
curl http://127.0.0.1/api/health
```

在你自己的电脑浏览器访问：

```text
http://服务器IP/api/health
```

看到类似下面内容就说明后端正常：

```json
{"ok":true,"service":"finance-payroll-server"}
```

## 10. 访问系统

浏览器打开：

```text
http://服务器IP
```

别人也可以通过这个地址访问测试。

## 11. 更新代码后重新发布

如果项目是 Git 拉取的：

```bash
cd /srv/finance-payroll-tool
git pull
docker compose -f docker-compose.prod.yml --env-file .env.production up -d --build
```

## 12. 停止服务

```bash
cd /srv/finance-payroll-tool
docker compose -f docker-compose.prod.yml --env-file .env.production down
```

## 13. 重要提醒

当前系统会处理工资、绩效等敏感数据。给别人试用前建议至少注意：

- 云服务器安全组只开放必要端口：`22`、`80`，后续有 HTTPS 再开放 `443`。
- 不要把 `.env.production` 发给别人。
- 测试阶段尽量上传脱敏数据。
- 正式上线前需要补登录权限和 HTTPS。
- 如果要长期保存文件，需要定期备份 Docker volume 里的 MySQL 和上传文件。
